How NTTH Works — End to End

The One-Line Summary

NTTH is an autonomous network defense system that captures network traffic from two sources (wired NIC + AR9271 WiFi adapter), scores every packet using AI (rules + ML), and automatically blocks attackers or redirects them to honeypots — all visible on a live Flutter dashboard.

What Happens When You Start the System

When you run sudo uvicorn app.main:app --port 8000, the system boots 8 concurrent tasks:

┌─ STARTUP SEQUENCE ────────────────────────────────────────────┐
│                                                                │
│  1. Database init (SQLite/PostgreSQL tables created)           │
│  2. Admin user seeded (admin / NtthAdmin2026)                  │
│  3. Event bus started (async pub/sub, 5000-event queue)        │
│  4. Scheduler started (periodic jobs)                          │
│  5. Wired packet sniffer started (Scapy on wlp0s20f3)    ←── IP packets
│  5b. WiFi sniffer started (Scapy on wlan0mon via AR9271) ←── 802.11 frames
│  6. HTTP honeypot started (port 8888)                          │
│  7. Cowrie log watcher started (tails cowrie.json)             │
│  8. Periodic network scan started (ARP every 60s)             │
│                                                                │
│  + 5 AI agents auto-subscribed to event bus at import time     │
│  + Flutter dashboard served at http://localhost:8000/           │
│  + REST API at http://localhost:8000/api/v1/                   │
│  + Swagger docs at http://localhost:8000/docs                  │
└────────────────────────────────────────────────────────────────┘

The Two Capture Pipelines

Pipeline A: Wired Traffic

YOUR LAPTOP NIC (wlp0s20f3)
       │
       ▼
  Scapy AsyncSniffer ──── filter: "ip" (all IP packets)
       │
       ▼
  feature_extractor.py ── extracts: src_ip, dst_ip, dst_port, protocol,
       │                  pkt_len, is_syn, is_ack, is_rst, flags, timestamp
       │
       ├──► device_registry.update()  ── per-IP state tracking
       ├──► update_live_stats()        ── real-time bandwidth
       │
       ▼
  EVENT BUS: "device_seen" ──────────────────────────────────────►

What it captures: Every IP packet on your WiFi network — web browsing, SSH, DNS, scans, attacks.

Pipeline B: Wireless Monitor Mode NEW — AR9271

AR9271 USB ADAPTER (wlan0mon) — monitor mode
       │
       ├── channel_hopper.py ── cycles channels 1–13 every 0.3s
       │
       ▼
  Scapy AsyncSniffer ──── no BPF filter (all raw 802.11 frames)
       │
       ▼
  wifi_feature_extractor.py ── parses 3 frame types:
       │
       ├── Probe Requests ──► probe_tracker.py ──► "wifi_probe_seen"
       │
       ├── Deauth Frames ──► deauth_detector.py ──► "wifi_threat_detected"
       │
       └── Beacon Frames ──► rogue_ap_detector.py ──► "wifi_threat_detected"

What it captures: Raw 802.11 management frames invisible to normal interfaces. Only possible because AR9271 supports monitor mode.

The Deception Layer

SSH Honeypot (Cowrie — Docker)

  • Port 30022 — accepts ANY credentials
  • Simulates fake Linux shell — logs all commands
  • Cowrie JSON log tailed in real-time by cowrie_watcher.py

Flow-Aware Redirection

  1. Detects brute force (3+ attempts in 120s)
  2. Creates nftables NAT rule redirecting ONLY that attacker to honeypot
  3. Attacker thinks they're in the real system — actually in the honeypot
  4. Commands captured and displayed on dashboard

Persistent Attacker Tracker NEW

Tracks attackers by MAC address — survives IP changes, DHCP reassignment, and reconnections:

ATTACK HAPPENS → MAC fingerprint stored
       │
ATTACKER DISCONNECTS... hours/days pass...
       │
ATTACKER RECONNECTS (new IP, same MAC)
       │
├── ARP scan finds device → MAC lookup → "KNOWN ATTACKER!" ⚠️
└── AR9271 detects WiFi probe → "ATTACKER NEARBY!" ⚠️ (before connection)

Deployment — 3 Options NEW

Option A: One-Command Docker Deploy

# Plug in AR9271 USB adapter, then:
cd /home/ubuntu/NTTH
docker compose up -d

# Monitor mode is auto-configured inside the container
# Dashboard: http://<your-ip>:8000/
# Health:    curl http://localhost:8000/api/v1/system/health
📦 Docker handles everything: installs deps, detects AR9271, enables monitor mode, starts backend, serves dashboard.

Option B: Auto-Start Script (Bare Metal)

# Plug in AR9271, then run:
sudo ./scripts/start_ntth.sh

# Auto-detects AR9271, kills conflicts, enables monitor mode, starts backend
# Optional flags:
#   --no-wifi    Skip wireless setup
#   --dev        Enable hot-reload
#   --port=9000  Change port

Option C: Manual (Step by Step)

1 Prepare the AR9271 (Manual)

sudo airmon-ng check kill
sudo airmon-ng start wlx24ec99bfe292
sudo airmon-ng   # Verify: wlan0mon ath9k_htc AR9271
💡 Auto-monitor: If you use Option A or B, this step is done automatically. The system detects AR9271 via lsusb, finds the interface, and enables monitor mode.

2 Start the Backend

cd /home/ubuntu/NTTH/backend
sudo uvicorn app.main:app --host 0.0.0.0 --port 8000 --log-level info

3 Verify System Health

curl http://localhost:8000/api/v1/system/health | python3 -m json.tool

TOKEN=$(curl -s -X POST http://localhost:8000/api/v1/auth/login \
